# Alert Bot 2.0 - Crypto Trading Bot by Way2Million

Telegram-бот для отслеживания цен криптовалют с системой алертов и торговых коллов.

## 🚀 Основные возможности

### 📊 Алерты
- **Алерты по цене** - уведомление при достижении целевой цены
- **Алерты по проценту** - уведомление при изменении на заданный процент
- **Резкие изменения** - автоматические уведомления при скачках цены >5% за 15 минут
- **История алертов** - полное логирование всех сработавших алертов

### 💹 Торговые коллы
- **Открытие коллов** - Long/Short позиции с фиксацией цены входа
- **Закрытие коллов** - автоматический расчет PnL в зависимости от направления
- **Реальтайм мониторинг** - отслеживание текущего PnL активных позиций
- **Рейтинг трейдеров** - сравнение результатов всех пользователей

### 📈 Аналитика и статистика
- **Персональная статистика** - winrate, совокупный PnL, лучшие/худшие сделки
- **Статистика по символам** - детализация по каждому торговому инструменту
- **История за 90 дней** - все расчеты основаны на актуальном периоде
- **Рейтинговая система** - сортировка трейдеров по результатам

## 🛠 Технические особенности

- **SQLite база данных** - надежное хранение данных с индексами (используется pure Go драйвер `modernc.org/sqlite`)
- **Bitget & Bybit API** - получение актуальных цен и исторических данных
- **Автоматический мониторинг** - проверка цен каждые 60 секунд
- **Кликабельные ID** - удобное копирование идентификаторов
- **Graceful shutdown** - корректное завершение при остановке

## 📋 Команды

### Общие команды
- `/start` - список всех команд бота
- `/chatid` - показать Chat ID, User ID и Username

### Алерты
- `/addalert TICKER price|pct VALUE` - создать алерт. *TICKER* автоматически дополняется USDT, если не указана другая стейблкоин-пара. Бот также запоминает *рынок* (спот/фьючерсы), на котором был найден тикер, для последующих запросов.
  - Пример: `/add BTC price 50000` (эквивалентно `/add BTCUSDT price 50000`)
  - Пример: `/add ETHUSDT pct -10`
- `/alerts` - показать все активные алерты пользователя
- `/del ID` - удалить алерт по ID
- `/clearallalerts` - удалить все алерты
- `/history [число]` - история сработавших алертов (по умолчанию 10)

### Коллы (торговые сигналы)
- `/ocall TICKER [long|short] [deposit_percent] [sl PRICE]` - открыть колл. *TICKER* автоматически дополняется USDT, если не указана другая стейблкоин-пара. Бот также запоминает *рынок* (спот/фьючерсы), на котором был найден тикер. *deposit_percent* (число >= 0) указывает процент от депозита, задействованный в сделке. *sl PRICE* (число >= 0) устанавливает стоп-лосс. По умолчанию: `long`, `0%` депозита и без стоп-лосса.
  - Пример: `/ocall BTC long 40` (открыть лонг по BTC с 40% депозита)
  - Пример: `/ocall BTC long 40 sl 25000` (открыть лонг по BTC с 40% депозита и стоп-лоссом 25000)
  - Пример: `/ocall ETH short` (открыть шорт по ETH с 0% депозита)
- `/sl CALLID [price]` - установить или обновить стоп-лосс для активного колла. *price* (число >= 0) устанавливает новую цену стоп-лосса. Если *price* не указан, стоп-лосс устанавливается на цену открытия колла.
  - Пример: `/sl abc123de 25000` (установить стоп-лосс на 25000)
  - Пример: `/sl abc123de` (установить стоп-лосс в цену открытия колла)
- `/ccall CALLID [size]` - закрыть колл по ID. *size* (от 1 до 100) указывает процент от оставшегося размера колла для закрытия. По умолчанию закрывается 100%.
  - Пример: `/ccall abc12345` (закрыть полностью)
  - Пример: `/ccall abc12345 50` (закрыть 50%)
- `/mycalls` - показать активные коллы с текущим PnL, оставшимся размером и стоп-лоссом
- `/allcalls` - показать все коллы всех пользователей (сортировка по PnL) и оставшимся размером
- `/rush` - закрыть все свои активные коллы разом

### Цены
- `/p TICKER` - показать цену одного символа с изменениями за разные периоды. *TICKER* автоматически дополняется USDT, если не указана другая стейблкоин-пара.
  - Пример: `/p btc` (эквивалентно `/p BTCUSDT`)
- `/priceall` - показать цены всех токенов из ваших алертов и коллов

### Статистика
- `/callstats` - рейтинг трейдеров за последние 90 дней
- `/mycallstats` - персональная статистика коллов за 90 дней
- `/mytrades` - статистика по символам за 90 дней
- `/stats` - статистика по активным алертам

   размером 10% от депозита

   /limit ETH b 3500 30 xyz789gh

5. Посмотреть активные ордера:
   /myorders

6. Отменить лимитный ордер:
   /climit abc123de

1. При срабатывании стоп-лосса все связанные лимитные ордера автоматически 
   отменяются

2. При полном закрытии колла все связанные лимитные ордера также отменяются

3. Лимитные ордера проверяются при каждом обновлении цены в мониторинге

4. Для закрытия Long позиции используйте направление 's' (sell/short)

5. Процент при создании ордера на закрытие означает процент от размера

## ⚙️ Установка и настройка

### Требования
- Go 1.21+
- SQLite3

### Клонирование и сборка
```bash
git clone <repository-url>
cd alert-bot
go mod tidy
go build -o bin/alert-bot cmd/bot/main.go
```

### Переменные окружения (.env)
```env
BOT_TOKEN=your_telegram_bot_token

# Bybit API Keys (опционально, если используется Bybit)
BYBIT_API_KEY=your_bybit_api_key
BYBIT_SECRET=your_bybit_api_secret

# Опциональные настройки
LOG_LEVEL=info
DATABASE_PATH=data/alerts.db
SHARP_CHANGE_PERCENT=10
SHARP_CHANGE_INTERVAL_MIN=15
```

### Запуск
```bash
# Обычный запуск
./bin/alert-bot

# В фоновом режиме
nohup ./bin/alert-bot &

# Через systemd
sudo systemctl start alert-bot
```

## 🗄️ Структура базы данных

### Таблица `alerts`
Хранит пользовательские алерты:
- `id` - уникальный короткий идентификатор (8 символов)
- `chat_id`, `user_id`, `username` - информация о пользователе
- `symbol` - торговый символ (BTCUSDT, ETHUSDT, etc.)
- `target_price` - целевая цена для алерта
- `target_percent`, `base_price` - процентное изменение и базовая цена
- `created_at` - время создания
- `market` - рынок (spot/futures), на котором был создан алерт

### Таблица `calls`
Хранит торговые коллы:
- `id` - уникальный идентификатор колла
- `user_id`, `username`, `chat_id` - информация о пользователе
- `symbol` - торговый символ
- `direction` - направление (long/short)
- `entry_price`, `exit_price` - цены входа и выхода
- `pnl_percent` - результат в процентах
- `status` - статус (open/closed)
- `size` - оставшийся размер колла (от 0 до 100)
- `market` - рынок (spot/futures), на котором был открыт колл
- `opened_at`, `closed_at` - временные метки

### Таблица `alert_triggers`
История сработавших алертов:
- Все сработавшие алерты и резкие изменения цен
- Используется для команд `/history` и аналитики

### Таблица `price_history`
Логи цен для аналитики:
- Сохраняются цены всех отслеживаемых символов
- Используется для исторического анализа

## 🔧 Особенности реализации

### Мониторинг цен
- Проверка каждые 60 секунд для всех символов с активными алертами или коллами
- Автоматическое добавление/удаление символов при создании/закрытии позиций
- Уведомления о резких изменениях для всех заинтересованных пользователей

### Расчет PnL для коллов
```go
// Long позиция
pnl = ((exitPrice - entryPrice) / entryPrice) * 100

// Short позиция  
pnl = ((entryPrice - exitPrice) / entryPrice) * 100
```

### Алерты по проценту с погрешностью
- Алерты по цене срабатывают с погрешностью ±0.5%
- Это исключает ложные срабатывания при незначительных колебаниях

### Кликабельные ID
Все идентификаторы в сообщениях обрамлены обратными кавычками:
```markdown
Алерт создан (ID: `abc12345`)
```
При нажатии на ID он автоматически копируется в буфер обмена.

## 🛡️ Безопасность и надежность

### Graceful Shutdown
Бот корректно обрабатывает сигналы остановки (SIGTERM, SIGINT):
- Завершает активные запросы к API
- Сохраняет все данные в базу
- Закрывает соединения

### Обработка ошибок
- Все внешние API вызовы имеют таймауты
- Повторные попытки при временных сбоях
- Детальное логирование всех ошибок

### База данных
- Транзакционность операций
- Индексы для быстрых запросов
- Автоматические миграции при обновлениях

## 📊 Миграция данных

При обновлении с версии на JSON файлах:

```bash
# Сборка миграционного скрипта
go build -o bin/migrate cmd/migrate/main.go

# Запуск миграции
./bin/migrate
```

Скрипт автоматически:
- Переносит все алерты из `alerts.json` в SQLite
- Создает backup старого файла
- Настраивает индексы и структуру БД

### Структура проекта
```
alert-bot/
├── cmd/
│   ├── bot/main.go          # Основное приложение
│   └── migrate/main.go      # Миграционный скрипт
├── internal/
│   ├── alerts/storage.go    # Работа с базой данных
│   ├── bot/bot.go           # Логика Telegram бота
│   ├── config/config.go     # Конфигурация
│   └── prices/              # Получение цен с Bitget и Bybit API
├── data/                    # База данных SQLite
└── README.md
```

### Добавление новых команд
1. Добавить обработчик в `handleUpdate()` в `bot.go`
2. Реализовать команду как метод `cmd[CommandName]()`
3. При необходимости добавить SQL запросы в `storage.go`
4. Обновить справку в `/start` и README
